IoT-Based Patient Monitoring System

Healthcare Tech ESP32 Machine Learning

Problem Statement

Hospital monitoring systems require continuous manual observation, creating staff workload and delayed response to critical patient conditions. Existing commercial solutions are prohibitively expensive for resource-limited healthcare facilities.

Solution & Technical Approach

Designed and deployed a low-cost IoT monitoring system integrating MAX30102 pulse oximeter and DHT22 sensors with ESP32 microcontroller. The system continuously tracks heart rate, SpO2, and body temperature, transmitting data via MQTT to a cloud dashboard for real-time visualization.

Implemented predictive anomaly detection using a Random Forest classifier trained on 5,000+ patient data points, achieving 94% accuracy in identifying critical health events 3-5 minutes before conventional alarms.

My Role: End-to-end system architect. Designed PCB layout, developed firmware in C++, built ML pipeline in Python, and deployed dashboard using Node-RED.

System Demonstration

Full system walkthrough: hardware setup, sensor calibration, real-time monitoring, and alert mechanisms

Technology Stack

Hardware: ESP32-WROOM-32, MAX30102, DHT22, custom PCB
Firmware: C++, Arduino framework, WiFi/MQTT libraries
ML Pipeline: Python, scikit-learn, pandas, matplotlib
Backend: MQTT broker (Mosquitto), Node-RED
Visualization: Grafana dashboard, real-time charts
Tools: KiCad (PCB design), Git, VS Code

Results & Impact

  • Deployed in university medical center for 30-day pilot study
  • Reduced false alarm rate by 68% compared to threshold-based systems
  • System cost: $35 per unit vs. $800+ for commercial alternatives
  • Average latency: 1.2 seconds from sensor reading to dashboard update
  • Published findings in IEEE RAAICON 2025 conference
